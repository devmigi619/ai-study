# 3단계 파이프라인

챗봇 응답을 3단계 LLM 호출로 고정하는 구조. 시도 중.

## 구조

1. **1차 LLM — 의도 파악.** 사용자 발화에서 의도·도메인을 파악한다. 응답 생성은 절대 금지. 모든 컨텍스트를 그대로 2차에 넘긴다.
2. **2차 LLM — ReAct 도구 호출.** 도구를 호출해 맥락을 수집한다. 도구가 필요 없으면, 그걸 확인하는 것도 2차의 역할이다. 빈 통과도 통과다.
3. **3차 LLM — 응답 생성.** 1·2차 결과를 받아 응답만 만든다.

## 왜

- LLM 호출이 무한정 쌓이면 상태 유실이 생기고, 지연이 누적되고, 어느 단계에서 잘못됐는지 특정하기 어렵다.
- 3단계로 고정하면 각 단계가 명확한 계약을 가진다. 단계별로 평가할 수 있다.
- 1차에 "쉬운 건 네가 응답해"라고 역할을 주면 LLM이 혼란에 빠진다. 역할을 명확히 주면 구조가 흔들리지 않는다.

## 대가

"안녕하세요" 같은 단순 질의도 3단계를 탄다 — 각 단계가 빠르게 통과할 뿐, 단축하지 않는다. 이건 손실이다. 그러나 다른 모든 대화에서 얻는 구조 안정성이 그 손실을 훨씬 덮는다.
