# 추천 플러그인

코딩에이전트와 함께 쓰는 플러그인 두 개. 말과 코드를 짧게 잡아준다. <u>[쉬운 말로 쓴다](./plain-language.md)</u>와 같은 뿌리 — 덜 쓰면서 더 정확하게.

## caveman — 말을 짧게

에이전트가 뱉는 말을 짧게. filler를 빼고 본질만 남긴다. 기술 정확도는 그대로, 출력 토큰은 크게 줄어든다.

https://github.com/juliusbrussee/caveman

예시:

> **Before**: "React 컴포넌트가 계속 다시 렌더링되는 이유는 매 렌더링마다 새로운 객체 참조를 만들고 있기 때문일 가능성이 높습니다. 인라인 객체를 prop으로 넘기면 React의 얕은 비교가 매번 다른 객체로 인식해서 다시 렌더링을 트리거합니다. `useMemo`로 객체를 메모이제이션하는 걸 권장합니다."
>
> **After**: "매 렌더링마다 새 객체 참조. 인라인 객체 prop = 새 참조 = 재렌더링. `useMemo`로 감싸기."

## ponytail — 코드를 간결하게

에이전트가 짠 코드를 간결하게. "게으른 시니어"처럼 먼저 물어본다: 이 코드가 필요한가? 표준 라이브러리에 있지 않은가? 플랫폼 기본 기능으로 안 되는가? 한 줄로 되지 않는가? 그 다음에야 최소한을 쓴다. 검증·보안·접근성은 자르지 않는다.

https://github.com/DietrichGebert/ponytail

예시:

> **Before**: 날짜 선택기를 만들려고 `flatpickr`를 설치하고, wrapper 컴포넌트를 쓰고, 스타일시트를 추가하고, 시간대 논의를 시작한다.
>
> **After**: `<input type="date">`
