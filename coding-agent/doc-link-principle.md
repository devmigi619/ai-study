# 문서는 링크구조로 만든다

코딩에이전트가 생성하는 문서는 항상 링크 구조로 작성하고 상호 참조시킨다. 단, 말단 문서(더 이상 하위로 갈라지지 않는 최종 내용)는 예외로 한다.

## 이유

- 단일 대형 문서는 에이전트가 전체를 파악하기 어렵고, 부분 수정 시 컨텍스트 오염 가능성이 크다.
- 링크 구조는 각 문서가 작고 독립적이므로 수정 범위가 제한되고, 참조 무결성을 유지하기 쉽다.
- 사람이 읽을 때도 탐색이 쉽다.

## 규칙

- 상위 문서는 반드시 하위 문서로의 링크를 포함한다.
- 하위 문서는 필요한 경우 상위·형제 문서를 참조한다.
- 말단 문서(leaf)는 더 이상 쪼갤 내용이 없는, 완결된 하나의 주제를 담는다.
