# TDD는 기본이다

코딩에이전트를 개발할 때 TDD(Test-Driven Development)를 기본 작업 방식으로 삼는다.

## 이유

에이전트는 비결정적 행동을 보이기 때문에, 출력을 사후 검증할 수 있는 테스트가 없으면 개발자가 에이전트의 품질 변화를 감지할 수 없다.

## 적용

- 기능 구현 전에 실패하는 테스트를 먼저 작성한다.
- 테스트는 가능한 한 결정적(deterministic)이어야 한다. LLM 출력에 의존하는 테스트는 범위를 좁게 잡고 평가 항목을 구체화한다.
- 회귀 방지를 위한 테스트셋을 지속적으로 확장한다.
