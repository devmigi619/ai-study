# 대화 데이터셋 적재가 먼저다

챗봇을 구현할 때는 대화 데이터셋을 적재하고 평가 파이프라인을 구축하는 것을 최우선 작업으로 삼는다.

## 이유

- 대화 데이터셋이 없으면 챗봇의 응답 품질을 평가할 기준이 없다.
- 실제 사용자 대화 로그를 기반으로 한 데이터셋이 이상적이다. 없다면 합성 데이터라도 먼저 확보한다.
- 데이터셋은 개발 중 회귀 테스트로, 배포 후 품질 모니터링 기준으로 지속 활용된다.

## 적용

- 초기 데이터셋은 도메인 전문가와 협의하여 구축한다.
- 실제 트래픽이 쌓이면 주기적으로 데이터셋을 갱신한다.
