# 도구 계약은 사용자와 협의한다

챗봇이 사용하는 도구(API, 함수, 검색 등)의 계약(입출력 형식·의미·제약)은 사용자(도메인 전문가·기획자·클라이언트)와 긴밀히 협의하여 결정한다.

## 이유

- 도구의 출력을 해석하는 방식은 도메인 지식에 의존한다. 개발자 혼자 판단하면 실제 사용 맥락과 어긋날 위험이 크다.
- 도구 계약은 곧 챗봇이 "무엇을 할 수 있는지"의 경계이므로, 제품 책임 범위와 직결된다.

## 적용

- 각 도구의 입력 파라미터 의미, 출력 필드의 해석 방식, 실패 시 처리 방침을 사용자와 합의한다.
- 합의 결과는 문서화하여 리뷰 에이전트와 평가 체계의 기준으로 삼는다.
- 도구 계약 변경은 반드시 사용자 승인을 거친다.
