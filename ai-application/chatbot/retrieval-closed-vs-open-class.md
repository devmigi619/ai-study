# 닫힌 분류는 코드로, 열린 의미는 LLM으로 (검색)

[자연어 사전 금지](./nl-dictionary-ban.md)와 [자연어 이해 방법론](./nl-understanding-methodology.md)의 검색(retrieval) 측면 정밀화. 제주 맛집 챗봇 PoC 의 측정으로 도달했다.

## 측정 요약

retrieve 벤치(현행 사전 ranking / 벡터 only / 사전=recall 전용 등 arm 비교, 임베딩 3종, LLM-judge 블라인드, 질의 22개)에서 사전의 효과가 질의 종류에 따라 정반대로 갈렸다.

## 경계

검색 의도를 두 종류로 가른다.

- **닫힌 분류 슬롯** — 음식 카테고리, 지역처럼 값이 유한하고 열거 가능하며 극성(긍정/부정)이 없는 속성. 여기서는 사전(동의어/택소노미 매핑)이 실제로 이긴다. 임베딩이 좋아져도 `해산물→생선회` 같은 짧은 카테고리 라벨의 어휘격차를 잘 못 넘기 때문이다.
- **열린 의미** — 상황, 분위기, 감정, 부정/역설처럼 변이가 무한하고 맥락·극성에 의존하는 의미. 여기서는 사전이 해친다. 키워드 존재만으로 의미를 정하므로 "웨이팅 없이"에 웨이팅 있는 곳을, "혼자 별로"를 "혼자 좋다"와 똑같이 가점한다. 임베딩이 좋아질수록 LLM+벡터가 사전을 더 확실히 앞선다.

또한 사전의 닫힌-분류 가치는 후보를 *건져오는 recall* 이 아니라 *순위를 매기는 ranking* 에 있었다. 사전을 recall 보조로만 강등하면 그 가치도 사라진다.

## 적용 — LLM 플래너

이해는 LLM 이 전담하되, 코드는 닫힌 택소노미 대조에만 쓴다.

1. LLM 플래너가 질의 원문을 구조화 계획으로 바꾼다.
   - 닫힌 슬롯(카테고리·지역): 데이터의 실제 택소노미로 **매핑**한다. 손으로 쓴 키워드/감정 리스트가 아니라, LLM 이 매핑해 들어가는 닫힌 분류 집합이다.
   - 열린 의미: 자연어 그대로 벡터 검색에 넘긴다.
   - 부정·제약(예: 웨이팅 없이, 주차 필수): 필터로 추출한다. 키워드 사전이 끝내 못 하던 일이다.
2. 닫힌 슬롯은 결정론 필터/부스트, 열린 의미는 벡터, 최종은 근거(리뷰 포함) rerank.
3. 플래너 출력은 반드시 실제 택소노미에 대조해 검증한다(할루시네이션 카테고리 차단). 이것이 "코드는 검증에" 의 구체형이다.

즉 사전은 사라지지 않고 변신한다 — 키워드/감정 사전(폐기)에서 "LLM 이 매핑하는 닫힌 택소노미"(생존)로.

## 한계

- 단일 PoC, 질의 22개. 방향성이지 통계적 유의성이 아니다.
- 벤치는 임베딩 텍스트에서 태그 줄을 제거해 비교했으므로, 닫힌-분류에서 벡터가 다소 불리하게 측정됐을 수 있다(사전 우위가 일부 과장).
- 플래너는 검색 앞단에 LLM 호출 1회를 더하므로 지연 예산(응답 시간 제약)과 함께 판단한다.

## 검증됨 — 플래너가 사전·벡터 둘 다 이긴다

위 아키텍처를 실제 arm 으로 벤치했다(bge, 질의 22개, 블라인드 judge): **P(플래너) 평균 7.41(10승) > A(현행 사전) 6.36(7승) ≈ B(벡터) 6.32(5승).** 버킷별로 P 가 닫힌-분류(dict_hit 8.0, 벡터 B 는 3.0)에서 사전만큼, 열린의미(situation 9.0)에서 벡터만큼, 부정(polarity, exclude 로 '웨이팅 없이' 처리)에서 둘 다 능가했다. 즉 "이해는 LLM, 코드는 닫힌-분류 검증"이 측정으로 최선임이 확인됐다. 수치는 [실측 요약](./dict-benchmark-evidence.md).
